PROBLEMA

A gerência de um estabelecimento da área de alimentação, requer uma gerência, complexa, pelo fato de ser vários setores se conversando e trocando informações entre si, de modo que um depende totalmente do bom desempenho do outro.

O fator Humano requer um treinamento adequado, claro e objetivo, de modo que todos os integrantes da equipe saibam suas funções, direitos e deveres, já o fator Sistema que manipula, gerencia as informações produzidas pelas equipes, requer um ajuste fino, adaptado e adequado a cada setor, em relação aquela empresa em questão, pois o funcionamento de cada empreendimento independente o porte e o tamanho funciona de maneira única.

A análise das funções básicas nos trás ao problema de gerência de estoque, precificação dos produtos, de maneira a garantir o funcionamento saudável do negócio.

OBJETIVO

Possibilitar ao administrador, e sua equipe, uma ferramenta capaz de auxiliar, em funções que devem ser automatizadas, fazendo assim que os membros das equipes desempenhar as funções sabendo que há um sistema para atende-los dando suporte aos mesmos.

garantir que haja uma base automatizada para cálculos internos, do cadastro da nota fiscal com a segurança de que não haja erro de valores, pois internamente há fatores que impossibilitam o erro, trazendo segurança e transparência, e evitando problemas no calculo financeiro posteriormente.

A Precificação dos produtos é baseada em porcentagem aplicadas internamente, no ato do cadastro de produtos(matéria prima para particionamento em ingredientes) e novamente na união dos ingredientes em itens disponíveis no cardápio valorizando assim o trabalho das equipes envolvidas.

REGRAS DE NEGÓCIO

1.Fornecedor

  • 1.1 - Os campos não podem ser nulos
  • 1.2 - A data de início de contrato deve ser inserida no formato: dd/mm/aaaa exemplo: 10/02/2024
  • 1.3 - O campo CNPJ não deve conter letras e caracteres especiais
  • 1.4 - O campos Telefone não deve conter letras e caracteres especiais
  • 1.5 - Os campos Email deve ser um email válido para contato
  • 1.6 - Os campos CEP deve conter apenas números
  • 1.7 - As datas de controle são obtidas automaticamente pelo sistema

2.Nota Fiscal

  • 2.1 - Os campos não podem ser nulos
  • 2.2 - O Fornecedor deve ter sido inserida no banco de dados para que o processo seja concluido comn sucesso
  • 2.3 - O valor da nota deve ser inserida integralmente, sendo lançada o valor relacionada a cada produto posteriormente
  • 2.4 - No valor da nota fiscal há um controle que garante o lançamento integral do valor cadastrado
  • 2.5 - O campo Valor Nota Fiscal não deve conter letras e caracteres especiais
  • 2.6 - O campos Numero Nota Fiscal não deve conter letras e caracteres especiais
  • 2.7 - As Datas de controle são obtidos automaticamente

3.Produto

  • 3.1 - Os campos não podem ser nulos
  • 3.2 - A Nota Fiscal deve ter sido inserida no banco de dados para que o processo seja concluído com sucesso
  • 3.3 - O valor da nota fiscal relacionada ao produto deve ser individual para cada item, para que o processo aconteça corretamente
  • 3.4 - O campo Nome deve ser breve e objetivo
  • 3.5 - O campo Descrição deve ser breve e objetivo
  • 3.6 - O campos Quantidade está relacionado de acordo com a unidade de medida selecionada, adequada a cada produto
  • 3.7 - O campos Porcentagem é a primeira adição de valor, ela não pode ser alterada posteriormente em relação a nota fiscal de entrada do produto, podendo ser alterado somente com o lançamento de uma nova nota fiscal
  • 3.8 - As Datas de controle são obtidos automaticamente

4.ingredientes

  • 4.1 - Os campos não podem ser nulos
  • 4.2 - O produto/Matéria prima deve ter sido inserida no banco de dados para que o processo seja concluído com sucesso
  • 4.3 - O campo Nome deve ser breve e objetivo
  • 4.4 - O campo Descrição deve ser breve e objetivo
  • 4.5 - O campos Quantidade está relacionado com o estoque do ingrediente
  • 4.6 - O campos Quantidade de produto está relacionado com a quantidade necessária para o preparo do ingredientes em "g" quando a unidade de medida selecionada na entidade produto for "KG" e "ml" em "L"
  • 4.7 - Há cálculos automatizados para fazer a redução automática no estoque do produto
  • 4.8 - As Datas de controle são obtidos automaticamente

5.Item Cardápio

  • 5.1 - Os campos não podem ser nulos
  • 5.2 - O ingrediente deve ter sido inserido no banco de dados para que o processo seja concluído com sucesso.
  • 5.3 - É possível utilizar vários ingredientes para criação do item, todos devem existir no banco de dados para que o processo seja concluído com sucesso.
  • 5.4 - O campo Nome deve ser breve e objetivo
  • 5.5 - O campo Descrição deve ser breve e objetivo
  • 5.6 - O campos Quantidade reservada é preenchido automaticamente pelo sistema
  • 5.7 - A porcentagem aplicada nesta fase leva em conta o custo do preparo dos ingredientes e o lucro desejado no produto final disponível para venda
  • 5.8 - Há cálculos automatizados para fazer a redução automática no estoque do ingrediente, assim como reserva garantindo que ao pedir um item tenha os ingredientes necessários
  • 5.9 - As Datas de controle são obtidos automaticamente

6.Atendimento

  • 6.1 - Os campos não podem ser nulos
  • 6.2 - Os itens adicionados devem ter sido inserido no banco de dados para que o processo seja concluído com sucesso.
  • 6.3 - É possível adicionar vários itens ao pedido, todos devem existir no banco de dados para que o processo seja concluído com sucesso.
  • 6.4 - O número da Mesa deve estar disponivel para que o processo seja concluído com sucesso sem transtornos, pois esta informação é gerenciada pelo fator humano não pelo sistema.
  • 6.5 - O atendimento é dividido em x status: coletado, em processo, pronto, entregue a mesa, finalizado, cancelado
  • 6.5.1 - Coletado: registrado ficando disponível para o em processo, reservando verificando e reservando a quantidade necessária de itens de cardápio
  • 6.5.2 - Em processo: retirando da quantidade reservada, e iniciando o processo de confecção
  • 6.5.3 - Pronto: pedido pronto para ser levado a mesa do cliente
  • 6.5.4 - Entregue a mesa: pedido entregue e pronto para ser finalizado
  • 6.5.5 - Finalizado: finalizando o atendimento mediante pagamento, liberando a mesa para novos atendimentos
  • 6.5.5 - Cancelado: Finaliza forçadamente o atendimento, de maneira que se o atendimento estiver em estágio de coletado, a quantidade reservada é retirada
  • 6.8 - Há cálculos automatizados para fazer a redução automática no estoque do ingrediente, assim como reserva garantindo que ao pedir um item tenha os ingredientes necessários
  • 6.9 - As Datas de controle são obtidos automaticamente

Tecnologias

Api Rest

Desenvolvido no IntelliJ IDEA Community Edition 2023.1.3, Utilizando Java Spring boot versão 3.2.2, na arquitetura clean architecture, garantindo uma segurança das informações nos processos de requisição, usando o MYSQL como Tecnologias de banco de dados.

Sistema de Segurança JWT ativo e funcional, impedindo que acessos indevidos sejam feitos, tanto por pessoas não registradas como registradas no sistema, cada equipe deve acessar somente oque é de sua competência, fazendo assim o bom funcionamento dos processos.

Docker disponibilizado para permitir teste e uso via container, de maneira a garantir o funcionamento ideal em qualquer ambiente, sendo possível também o deploy do mesmo nos mecanismos apropriados, pois a configuração está desenvolvida e pronta para uso.

A página de documentação foi desenvolvida em HTML 5, CSS 3, interligada a Api Rest, via thymeleaf com conteúdo estático, pela razão de não haver grandes alterações constantes não havendo a necessidade uma inserção e manutenção de dados dinâmica.
